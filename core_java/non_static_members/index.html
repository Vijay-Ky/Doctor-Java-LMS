Non-Static Members in Java

Introduction

Non-static members in Java are those members that belong to individual instances of a class rather than the class itself. They are also known as instance members because they are associated with objects (instances) of the class. Each object gets its own copy of non-static members.

Non-Static Variables (Instance Variables)

Non-static variables are declared without the static keyword. They are created when an object is instantiated and destroyed when the object is garbage collected. 

  class Student {
      // Non-static variable (instance variable)
      String name;
      int rollNo;
      
      // Constructor
      Student(String n, int r) {
          name = n;
          rollNo = r;
      }
      
      void display() {
          System.out.println("Name: " + name + ", Roll No: " + rollNo);
      }
  }
  
  public class Main {
      public static void main(String[] args) {
          // Creating objects
          Student s1 = new Student("Alice", 101);
          Student s2 = new Student("Bob", 102);
          
          // Each object has its own copy of instance variables
          s1.display(); // Name: Alice, Roll No: 101
          s2.display(); // Name: Bob, Roll No: 102
      }
  }

Key characteristics of non-static variables:

  • They are created when an object is created with new
  • Each object gets its own copy of non-static variables
  • They can be accessed only through object reference
  • They exist as long as the object exists
  • They are stored in heap memory

Non-Static Methods (Instance Methods)

Non-static methods are methods that belong to objects of the class. They can access both static and non-static members of the class. 

  class Calculator {
      // Non-static method
      int add(int a, int b) {
          return a + b;
      }
      
      // Another non-static method
      int multiply(int a, int b) {
          return a * b;
      }
  }
  
  public class Main {
      public static void main(String[] args) {
          Calculator calc = new Calculator();
          
          // Calling non-static methods using object reference
          int sum = calc.add(5, 3);
          int product = calc.multiply(5, 3);
          
          System.out.println("Sum: " + sum);       // Sum: 8
          System.out.println("Product: " + product); // Product: 15
      }
  }

Key characteristics of non-static methods:

  • They can access both static and non-static members directly
  • They must be called using an object reference
  • They can use this keyword to refer to current object
  • They can be overridden in subclasses

Non-Static Blocks (Instance Initialization Blocks)

Non-static blocks are used to initialize non-static variables. They are executed each time an object is created, before the constructor. 

  class Demo {
      // Non-static variable
      int x;
      
      // Non-static block
      {
          System.out.println("Non-static block executed");
          x = 10;
      }
      
      // Constructor
      Demo() {
          System.out.println("Constructor executed");
          System.out.println("x = " + x);
      }
  }
  
  public class Main {
      public static void main(String[] args) {
          Demo d1 = new Demo();
          Demo d2 = new Demo();
      }
  }
  
  /* Output:
  Non-static block executed
  Constructor executed
  x = 10
  Non-static block executed
  Constructor executed
  x = 10
  */

Key points about non-static blocks:

  • They are executed for each object creation
  • They run before the constructor
  • Multiple blocks are executed in the order they appear in the code
  • They are useful when you need to perform common initialization for all constructors

Initialization of Non-Static Members

Non-static members can be initialized in several ways:

  1. Direct initialization at the point of declaration
  2. Through constructors
  3. Using non-static initialization blocks
  4. Through setter methods after object creation
  class Employee {
      // Direct initialization
      String department = "Unknown";
      
      // Initialized through constructor
      String name;
      
      // Initialized through non-static block
      int id;
      {
          id = (int)(Math.random() * 1000);
      }
      
      Employee(String n) {
          name = n;
      }
      
      void display() {
          System.out.println("ID: " + id + ", Name: " + name + ", Department: " + department);
      }
  }
  
  public class Main {
      public static void main(String[] args) {
          Employee e1 = new Employee("John");
          e1.display();
          
          Employee e2 = new Employee("Sarah");
          e2.department = "HR"; // Initialized through setter
          e2.display();
      }
  }

Memory Allocation for Non-Static Members

Non-static members are allocated memory when an object is created using the new keyword. They are stored in the heap memory area. 

  class MemoryDemo {
      int x = 10;  // Non-static variable (in heap)
      
      void show() {
          int y = 20;  // Local variable (in stack)
          System.out.println("x = " + x + ", y = " + y);
      }
  }
  
  public class Main {
      public static void main(String[] args) {
          MemoryDemo d1 = new MemoryDemo();  // Allocates memory for x
          MemoryDemo d2 = new MemoryDemo();  // Allocates separate memory for x
          
          d1.x = 100;  // Modifies x for d1 only
          
          d1.show();  // x = 100, y = 20
          d2.show();  // x = 10, y = 20
      }
  }

Memory allocation process:

  1. When new MemoryDemo() is executed, memory is allocated in heap
  2. All non-static variables are created in this memory space
  3. Each object gets its own separate copy of non-static variables
  4. Non-static methods are shared among all objects (stored in method area)

Non-Static vs Static Members

Feature Non-Static Members Static Members
Keyword No special keyword Uses static keyword
Memory Allocation When object is created When class is loaded
Memory Location Heap memory Method area
Access Through object reference Through class name or object reference
Copy Each object gets its own copy Only one copy shared by all objects
Lifetime As long as the object exists Entire program execution
Can access Both static and non-static members Only static members directly
Overriding Can be overridden Cannot be overridden (can be hidden) 
  class Example {
      int x;          // Non-static variable
      static int y;   // Static variable
      
      void method1() { // Non-static method
          x = 10;     // OK
          y = 20;     // OK
          method2();  // OK
      }
      
      static void method2() { // Static method
          // x = 10;         // Error - cannot access non-static member
          y = 20;            // OK
          // method1();      // Error - cannot call non-static method
      }
  }
